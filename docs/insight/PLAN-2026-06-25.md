### Unit A — ack-capability contract + SQS correct-URL ack  *(C1 CRITICAL + C3 HIGH)*
**Files**: `src/scrapy_extension/backends/base.py`, `src/scrapy_extension/backends/{kafka,rabbitmq,sqs,pulsar}.py`,
`src/scrapy_extension/schedule/scheduler.py`, `tests/test_sqs_backend.py`, `tests/test_scheduler_ack_gate.py` (new).

**A1 — capability contract** (`backends/base.py` `QueueBackend`):
- Add class attrs: `requires_ack: bool = False` (True = pop yields a message needing explicit ack;
  False = atomic pop, ack is a no-op) and `supports_concurrent_ack: bool = True` (False = single-slot,
  unsafe under `CONCURRENT_REQUESTS>1`).
- Defaults make atomic-pop backends (Redis/Mongo/ES/RocketMQ) pass untouched.

**A2 — per-backend declaration** (the 4 MQ backends):
- `KafkaQueueBackend`/`RabbitMQQueueBackend`: `requires_ack = True`, `supports_concurrent_ack = True` (real in-flight set).
- `SqsQueueBackend`/`PulsarQueueBackend`: `requires_ack = True`, `supports_concurrent_ack = False`.

**A3 — scheduler gate** (`schedule/scheduler.py` `from_settings`, after the queue backend is resolved):
- Read `CONCURRENT_REQUESTS` from Scrapy settings (default 16). If
  `type(backend).requires_ack and not type(backend).supports_concurrent_ack` and
  `CONCURRENT_REQUESTS > 1` and not `SCRAPY_ACK_UNSAFE_CONCURRENT_REQUESTS` (env/setting, default false)
  → raise `ConfigurationError` with a message naming the backend + the opt-out.
- Docstring: correct the over-claim at `scheduler.py:67-69` — state the gate's exact contract.

**A4 — SQS correct-URL ack** (`backends/sqs.py`, the C3 bug):
- Replace single `_last_receipt: str | None` with `_last_receipt: tuple[str, str] | None` =
  `(queue_url, receipt_handle)`. `pop` records the URL it received from; `ack` deletes against that
  URL. Drop `del queue_name`; keep `token` unused (signature compat) until the real in-flight-set lands.
- **Behavior change**: multi-queue SQS now acks correctly. Single-queue unchanged.

**Tests (TDD — RED first)**:
- `test_scheduler_ack_gate.py`: SQS+`CONCURRENT_REQUESTS=16` → `ConfigurationError`; opt-out flag →
  passes; Kafka/RabbitMQ+16 → passes (real ack); Redis+16 → passes (atomic); atomic+1 → passes.
- `test_sqs_backend.py`: pop from qB → `ack` calls `delete_message(QueueUrl=qB_url, ReceiptHandle=…)`,
  NOT qA_url (RED pre-A4, GREEN post).
