# SPEC — Round 2 Hardening (2026-06-25)

Companion to [`INSIGHTS-2026-06-25.md`](./INSIGHTS-2026-06-25.md) (evidence) and
[`PLAN-2026-06-25.md`](./PLAN-2026-06-25.md) (execution). Builds on round-1
[`SPEC.md`](./SPEC.md) — does not re-open closed items.

## Problem

Round-1 closed the single-backend surface. Round-2 insight surfaced **1 CRITICAL + 6 HIGH**
defects at the **inter-component & distributed seams** that survive the 1203-green bar. The
headline (C1): SQS/Pulsar ack is single-slot and the `CONCURRENT_REQUESTS>1` guard was removed,
so under default Scrapy settings (`CONCURRENT_REQUESTS=16`) the scheduler **silently** drops
15/16 acks while its docstring asserts correctness. The same silent-failure pattern recurs at the
pipeline (C2: storage outage swallowed as success), the batched flush (C4: partial-failure data
loss), and the dedup lifecycle (C5: resume crawls nothing).

## Goal

Close the CRITICAL + HIGH seam defects with **surgical, unit-verifiable** changes that preserve
the 95% coverage floor and the existing public API. Convert **silent** data-loss / false-success
into **loud, fail-fast or stat-visible** behavior — the same philosophy as round-1, applied to the
seam layer round-1 didn't reach.

## In-scope (Tier 1 — this pass)

1. **C1 — Capability-aware ack gate (CRITICAL).** Re-introduce a per-backend ack-capability
   contract (`QueueBackend.requires_ack` / `supports_concurrent_ack`) and a scheduler
   `from_settings` gate: MQ backend with single-slot ack + `CONCURRENT_REQUESTS>1` + no opt-out
   → `ConfigurationError`. Atomic backends and Kafka/RabbitMQ unaffected.
2. **C3 — SQS ack correct-queue resolution (HIGH).** Track `(queue_url, receipt_handle)` per pop;
   `ack` deletes against the source queue, not an arbitrary cached URL.
3. **C2 — Pipeline storage-error escalation (HIGH).** Additive `SCRAPY_PIPELINE_MAX_STORAGE_ERRORS`
   (default off → behavior preserved); when set, consecutive storage errors raise instead of
   swallowing. Wire `monitor.on_store` (closes F-asymmetry).
4. **C4 — Batched storage at-least-once flush (HIGH).** Don't clear `_buffer` before writes
   commit; on partial failure, re-enqueue the unwritten tail.
5. **C5 — Dedup lifecycle (HIGH).** Additive `SCRAPY_DUPEFILTER_CLEAR_ON_OPEN` (default off) +
   `{spider}` template support for dedup/queue keys; factory warning when a per-process dedup
   strategy is selected.
6. **C6 + C8 — Scheduler envelope & key templating (HIGH).** Move `dupefilter.request_seen` inside
   `enqueue_request`'s try-block; support `{spider}` substitution in `queue_key`/`dupefilter_key`.
7. **B2/B3 — Circuit-breaker hardening (MEDIUM→HIGH if probe-race confirmed).** Remove `queue_len`
   from `_HOT_PATH`; enforce single half-open probe under the lock.

## Non-goals (Tier 2/3 — deferred; spec'd in PLAN)

- Real in-flight-set ack for SQS/Pulsar (the gate is the honest minimal fix; real fix is follow-up).
- Backpressure action hook (consume `queue/backpressure` → throttle / `DOWNLOAD_DELAY`).
- Distributed `DelayQueueStrategy` (durable heap); distributed throttle token bucket.
- Per-process dedup strategies → distributed variants (Bloom/Cuckoo across workers).
- Entry-point plugin registration for backends.
- `_RedactedStr` parity across all backends; Mongo/ES/SQS TLS/creds mode guards (security-parity).
- Sentinel failover re-discovery; `rocketmq-client-python` replacement.

## Constraints

- **Public API stable** (component factories, settings field names, default behaviors). All new
  settings are **additive** with defaults that preserve current behavior.
- **TDD**: regression test first (RED, fails pre-fix) → minimal fix (GREEN) → refactor. Coverage
  ≥ 95% on changed lines; project floor 95%.
- **Immutability + no silent error swallowing** (project `CLAUDE.md`) — except where swallowing is
  the documented contract (pipeline best-effort), in which case the new escalation setting is the
  opt-in loud path.
- **Unit-verifiable without real backend services** (mocks; integration CI excludes RocketMQ).
- **Test honesty**: never disable/weaken/skip assertions to green a suite (tests/ is
  scoring-sensitive). Limitation-pinning `xfail`/`skip` with a reason is allowed (documents a known
  gap); grading-evaluation gaming is not.

## Acceptance (done when)

- `uv run pytest` fully green (existing 1203 + new regression tests), `-p no:randomly` stable.
- `uv run ruff check src tests` clean; `uv run mypy src/scrapy_extension` clean.
- Each Tier-1 defect has a regression test that **fails pre-fix, passes post-fix** (C3/C4/B3 RED
  tests especially — they confirm HYPOTHESIS-graded findings before the fix is trusted).
- No new public-API breakage (additive settings only; no default-behavior change unless the old
  default was a silent-defect vector, called out per-unit in PLAN).
- INSIGHT.md/PLAN.md round-1 doc-accuracy fixes applied (ES atomic pop → done; integration CI →
  resolved) — non-blocking, done last.
